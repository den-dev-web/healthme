# Техническое задание (ТЗ)
# **HealthMe — приложение-консультация врача**

---

## **1. Общая информация о проекте**
**Название:** HealthMe — мобильное веб-приложение для онлайн‑консультаций с врачами.  
**Тип:** mobile-first приложение с консультативным функционалом, карточками врачей, формой записи и UI-компонентами.

**Цель:** продемонстрировать навыки mobile-first дизайна, touch‑UX, адаптивных сеток, карточек специалистов, формы записи, навигации в стиле мобильного приложения, а также компонентного подхода.

**Стек:**
- HTML5 (семантика + a11y)
- CSS (ITCSS + BEM)
- Vanilla JavaScript
- WebP / AVIF изображения
- GitHub Pages / Vercel

Проект должен выглядеть как современное мобильное мед-приложение.

---

## **2. Структура страниц**
Одна основная страница: `index.html`.  
Опционально — страница врача: `doctor.html`.

### **2.1. Главная страница (index.html)**
Секции:
1. Header (стиль приложения)
2. Hero
3. Категории врачей
4. Список специалистов
5. Блок "Как это работает"
6. Отзывы пациентов
7. Форма записи на консультацию
8. Bottom Navigation (мобильная навигация)
9. Footer

### **2.2. Страница врача (doctor.html)
Секции:
- Фото врача
- Имя, специализация, опыт
- Рейтинг и отзывы
- Подробное описание
- Расписание (карточки времени)
- Форма записи на консультацию

---

## **3. Основной функционал**
### **3.1. Категории врачей**
- Горизонтальный скролл категорий
- Иконка + название
- По клику — фильтрация списка врачей

### **3.2. Список врачей**
Каждая карточка врача включает:
- Фото (круглое)
- Имя
- Специализация
- Опыт (например: «Опыт 8 лет»)
- Рейтинг
- Кнопка → «Записаться» или «Подробнее»

### **3.3. Фильтрация (JS)**
По клику категория → показываются врачи только данной категории.

### **3.4. Bottom Navigation**
Типичная навигация как в приложениях:
- Главная
- Врачи
- Запись
- Профиль (заглушка)

Должно ощущаться как настоящее приложение.

### **3.5. Форма записи**
Поля:
- Имя
- Телефон
- Категория врача
- Комментарий
- Чекбокс согласия

Функционал:
- Валидация полей
- Состояние отправки (loading)
- Отправка через fetch на mock endpoint
- aria-live сообщения

---

## **4. Адаптивность и mobile-first**
Проект строится **в первую очередь под мобильные устройства**.

### Breakpoints:
- <600px — мобильные (основной UX)
- 600–1024px — планшеты
- >1024px — десктоп (расширенный вид)

### Изменения:
- Карточки врачей: 1 → 2 → 3 колонки
- Bottom Navigation прячется на десктопе
- Layout становится более воздушным на больших экранах

---

## **5. Touch UX (обязательно)**
- Крупные кликабельные зоны (минимум 44px)
- Радиус кнопок 12–16px
- Естественные зоны взаимодействия (нижняя навигация, крупные карточки)
- Увеличенные интервалы между элементами
- Анимации при нажатии (сжатие, затем восстановление через transform)

---

## **6. Анимации**
- Scroll reveal для Hero, категорий и списка врачей
- Анимация нажатия кнопок (scale 0.97 → 1)
- Плавные hover/active состояния
- Открытие/закрытие формы
- Prefers-reduced-motion поддерживается

---

## **7. Доступность (a11y)**
- Семантика секций и заголовков
- Контрастные focus‑стили
- Управление формой с клавиатуры
- aria-live для ошибок и успехов
- alt-тексты у всех фото врачей
- Управляемая клавиатурой нижняя навигация

---

## **8. Оптимизация**
- Изображения в WebP/AVIF
- Lazy-loading фото врачей
- aspect-ratio для карточек
- Минификация CSS/JS
- Цель по Lighthouse: A11y — 100, Performance — 90+

---

## **9. Критерии готовности**
1. Полностью реализованный mobile-first UX.
2. Рабочая фильтрация врачей.
3. Карточки врачей с фото, рейтингом и опытом.
4. Bottom Navigation в стиле приложения.
5. Рабочая форма записи.
6. Анимации и touch‑эффекты.
7. Полная доступность с клавиатуры.
8. Оптимизация изображений и lazy-loading.
9. Публичный деплой.

---

## **11. Итог**
HealthMe должно выглядеть как современное мобильное мед-приложение с удобной навигацией, карточками врачей, формой записи, классным touch‑UX и анимациями.  
Проект подчёркивает твой уровень владения адаптивной версткой, мобильным UX, компонентами, формами и JS‑интерактивами.
